## 第 2 轮：添加衣物报错 — `iterator method is not callable`

### 测试操作
- 点 FAB "+" → 拍照 → AI 识别 → 编辑 → 保存

### 测试结果 ❌
App 报错崩溃：
```
TypeError: iterator method is not callable
at arraySpread (<native>)
at setItems$argument_0 (app\wardrobe.jsx)
```

### 原因分析
`loadWardrobe()` 中 `res.data` 可能不是纯数组。API 可能返回 `{ items: [...] }` 或 `{ data: {...} }` 格式。导致 `items` 状态被设置成非数组对象。后续 `handleAddSaved` 中 `[...prev]` 展开时报错。

### 修复
1. **wardrobe.jsx** — 新增 `ensureArray()` 归一化函数，适配 3 种 API 格式
2. **wardrobe.jsx** — 所有 `setItems` 回调中加 `Array.isArray(prev)` 防御
3. **AddItemModal.jsx** — `handleSave` 中归一化 POST 返回值：`result.data?.item || result.data?.data || result.data`

### 再次测试 ✅
- 拍照 → AI 识别 → 编辑 → 保存 → 列表出现新卡片 → 正常

## 第 3 轮：显示原图而非抠图

### 测试操作
- 批量上传 9 张图 → AI 识别成功（后端日志确认） → 保存

### 测试结果 ❌
手机上显示的是上传的原图，不是 AI 抠完图的处理后图片。

### 原因分析
`AddItemModal.jsx` 中 `handleSaveAll` 的 payload：
```js
processed_image: item.base64 || item.aiResult?.processed_image_base64 || null
```
`item.base64` 是原图且永远为真，所以永远不取 `aiResult.processed_image_base64`（抠图结果）。

### 修复
调换顺序，优先取抠图结果：
```js
processed_image: item.aiResult?.processed_image_base64 || item.base64 || null
```

### 再次测试 ⚠️
- 新上传的衣物显示抠图正常 ✅
- 之前保存的 9 件仍显示原图（已存后端数据无法更改）

## 第 4 轮：改批量导入

### 用户要求
> 选完一张就直接进识别了，改成多选后再统一识别

### 改造方案
将 AddItemModal 从单张流程改为批量流程：

| 步骤 | 旧流程 | 新流程 |
|------|--------|--------|
| 1. 选图 | 拍照/相册 → 立即识别 | 拍照（每次一张）/ 相册多选 → 积累队列 |
| 2. 识别 | 处理中... | "正在识别 2/5..." 逐张处理 |
| 3. 编辑 | 单张表单 | 列表折叠展开，每项独立编辑 |
| 4. 保存 | 保存一件 | "全部保存 (N件)" 逐件保存 |

关键改动：
- `launchImageLibraryAsync` + `allowsMultipleSelection: true`
- `pendingImages[]` 队列积累选图
- `processedBatch[]` 存储识别结果
- FlatList 折叠式逐项编辑
- `cancelledRef` 用于中断异步循环

### 测试
- 相册多选 9 张 → 已选 N 张预览横滑 → 点 "开始识别" ✅
- 进度显示 "正在识别 1/9" → 9/9 → 进入确认列表 ✅
- 展开单项编辑名称/分类/颜色 → 全部保存 ✅

## 第 6 轮：批量多选功能

### 用户要求
> 添加多选按钮，点开之后有批量删除、批量标记脏衣服、批量不要的功能

### 实现内容

**ClothingCard.jsx** — 新增 props：
- `multiSelect` — 是否多选模式
- `isSelected` — 是否已选中
- 视觉效果：左上角勾选框、选中时蓝紫色边框 + 淡紫遮罩
- 禁用按压动画、隐藏脏衣标记

**wardrobe.jsx** — 新增功能：
- `multiSelectMode` + `selectedIds` (Set) 状态
- Header 切换："多选" 按钮 → 进入模式后显示 "全选 / 已选 N 件 / 完成"
- 多选模式下隐藏搜索栏、分类筛选、FAB
- 底部批量操作栏：标记脏衣 / 标记不要 / 删除
- 3 个批量函数：
  - `batchMarkDirty()` — 本地立即更新 + 异步逐件调 API
  - `batchMarkUnwanted()` — 同上
  - `batchDelete()` — Alert 确认 → 乐观删除 → 全失败则回滚 → Alert 提示

### 测试
1. 点 "多选" → 进入多选模式，header 变化 ✅
2. 点若干卡片 → 已选 N 件，勾选 + 边框 ✅
3. "全选" → 全部选中 / "取消全选" → 全部取消 ✅
4. 底部 "标记脏衣" → 选中项即时标记，列表出现红点 ✅
5. 底部 "标记不要" → 选中项即时标记 ✅
6. 底部 "删除" → 确认弹窗 → 卡片消失 → 没有的（无报错）✅
7. "完成" → 退出多选，恢复正常模式 ✅

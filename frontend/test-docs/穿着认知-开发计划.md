## 二、页面概述

**核心玩法**：选一件衣服 → AI 匹配适合这件衣服的时尚人格 → 人格以第一人称给你穿搭建议。

**适用场景**：
- "我有件格子裙不知道怎么搭" → 匹配古着收藏家 → AI 给复古混搭思路
- "这件卫衣今天穿什么下装" → 匹配街头潮人 → AI 给潮流建议

## 四、交互流程

### 流程 A：衣物匹配人格 → AI 咨询

```
1. 点"选择衣物" → 从衣柜选一件（弹窗/底部 Sheet）
2. 选好后衣物标签显示在输入框上方
3. 点"帮我搭配" → 后台自动调用 matchPersonas + personaThink
4. 显示匹配结果："系统已为「格子裙」匹配到 古着收藏家、浪漫诗人..."
5. AI 以第一人称逐字输出回复（TypeWriter）
6. 可切换人格重新咨询
```

### 流程 B：直接选人格 → 自由提问（无衣物绑定）

```
1. 点击人格卡片 → 选中高亮
2. 输入框直接输入文字（如"明天约会穿什么好"）
3. 点发送 → personaThink(personaId, userProblem, '')
4. AI 以人格口吻回复
```

### 流程 C：快速咨询（自动匹配 + 预设问题）

```
1. 选中衣物 → 自动匹配人格
2. 显示预设快捷提问："这件怎么搭？""适合什么场合？"
3. 点击即发送 → AI 逐字回复
```

### 5.2 TypeWriter

**实现**：逐字显示动画。

```javascript
function TypeWriter({ text, speed = 50, onComplete }) {
  const [displayed, setDisplayed] = useState('');
  const index = useRef(0);

  useEffect(() => {
    index.current = 0;
    setDisplayed('');
    const timer = setInterval(() => {
      if (index.current < text.length) {
        setDisplayed(text.slice(0, index.current + 1));
        index.current++;
      } else {
        clearInterval(timer);
        onComplete?.();
      }
    }, speed);
    return () => clearInterval(timer);
  }, [text]);

  return <Text>{displayed}<Text style={{opacity: 0.4}}>|</Text></Text>;
}
```

## 七、API 调用时序

### 时序 1：选衣物 → 匹配 → AI 回复

```
1. fetchPersonas()                        → 获取 6 个人格数据
2. 用户选衣物 → selectedItem 更新
3. matchPersonas(tag, category)            → 获取匹配人格 ID 列表
4. 匹配结果展示在人格卡片上（星标标注）
5. 自动选第一个匹配人格（或用户手动选）
6. personaThink(personaId, userProblem, clothingTag) → 获取 AI 回复
7. TypeWriter 逐字展示
```

### 时序 2：直选人格 → 自由提问

```
1. fetchPersonas()                        → 获取人格
2. 用户选人格 + 输入文字
3. personaThink(personaId, userProblem, '') → 获取 AI 回复
4. TypeWriter 展示
```

## 十、边界情况

| 场景 | 处理 |
|------|------|
| 未选衣物直接点"帮我搭配" | 提示"请先选择一件衣物" |
| 未选人格发送消息 | 自动用"实用顾问"（默认） |
| AI 返回空 | "AI 暂时无法生成回复，请换个问法试试" |
| API 超时 (120s) | 显示超时提示 + 重试按钮 |
| 无网络 | 断网提示 |
| 衣物为"杂物"分类 | 跳过匹配，提示"杂物不适合穿搭咨询" |
| 快速连续点发送 | 发送中按钮置灰 + loading 禁用 |
| TypeWriter 进行中点发送 | 中断当前打字，开始新回复 |

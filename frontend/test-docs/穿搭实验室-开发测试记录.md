## 第 2 轮：图片不显示 — 缺失 base64 数据 URI 前缀

### 测试操作
- 进入穿搭实验室页面 → 观察衣物选择条
- 点击衣物缩略图 → 观察画布

### 测试结果 ❌
- 衣物选择条缩略图为空白/灰色占位
- 点击衣物后，画布上元素也为空白（或不可见）
- 后端 API `GET /api/wardrobe` 返回 200，数据正常

### 原因分析
后端 `/api/process` 和 `/api/process_portrait` 返回的 `processed_image_base64` 是**纯 base64 字符串**（如 `iVBORw0...`），不含 `data:image/png;base64,` 前缀。React Native 的 `<Image source={{ uri: 'iVBORw0...' }}>` 无法识别纯 base64 为合法 URI。

对比：已有的 `ClothingCard` 组件有 `getImageSource()` 辅助函数处理此转换，但新建的 `ootd-lab.jsx` 直接用了 `{ uri: item.processed_image }`。

### 修复
1. **ootd-lab.jsx** — 新增 `getImageSource(raw)` 助手函数（与 ClothingCard 逻辑一致）：
   - `!raw` → 返回 `null`
   - `raw.startsWith('data:')` 或 `raw.startsWith('http')` → 直接用作 URI
   - 否则 → 前置 `data:image/png;base64,`
2. `loadOutfit` 中 canvas 元素 `image` 字段 → 转换为完整 data URI
3. `toggleItem` 中 canvas 元素 `image` 字段 → 同上
4. FlatList `renderItem` 中衣物缩略图 → 改用 `getImageSource(item.processed_image)`

### 再次测试 ⚠️
衣物缩略图仍不显示（实际原因见第 3 轮）。

## 第 3 轮：AI 抠图结果不显示 — 双重 base64 前缀

### 测试操作
- 工具栏 → 点"📷 人脸" → 拍照 → 等待 AI 处理

### 测试结果 ❌
- 后端日志显示 200 成功
- 前端弹出"AI 抠图结果"预览框，但图片为空白
- 点击"使用"后，画布上人脸元素也为空白

### 原因分析
后端 `process_portrait` 端点（`app.py:181`）返回的 `processed_image_base64` **已经包含** `data:image/png;base64,` 前缀：

```python
# backend/app.py 第 181 行
base64_str = f"data:image/png;base64,{base64_encoded}"
```

而前端 `startFaceImport` 中又加了一次：

```javascript
// 旧代码（双重前缀）
setPortraitImage(`data:image/png;base64,${processed}`);
```

结果形成 `data:image/png;base64,data:image/png;base64,iVBORw0...`，Image 组件无法解析。

### 修复
**ootd-lab.jsx** `startFaceImport` — 添加 `startsWith('data:')` 守卫判断：

```javascript
setPortraitImage(
  processed.startsWith('data:') ? processed : `data:image/png;base64,${processed}`
);
```

### 再次测试 ✅
- AI 抠图结果预览正常显示 ✅
- 点击"使用"后，人脸元素正常出现在画布上 ✅

## 第 4 轮：空衣柜 loading 永远不消失

### 测试操作
- 清空衣柜所有衣物 → 进入穿搭实验室页面

### 测试结果 ❌
页面永远显示加载 Spinner + "加载衣柜数据..."，无法进入正常操作界面。

### 原因分析
原代码 `useEffect` 的守卫条件：

```javascript
useEffect(() => {
    if (allItems.length > 0) {   // ← 空衣柜时此处永远为 false
      setLoading(false);
      loadOutfit(selectedDate);
    }
  }, [allItems, selectedDate]);
```

当 API 返回空数组 (`[]`) 时，`allItems.length` 为 0，`setLoading(false)` 永远不会被调用。

### 修复
1. 新增 `wardrobeLoaded` (boolean) 状态，在 `loadWardrobe()` 的 `finally` 中设置为 `true`
2. `useEffect` 改为监听 `wardrobeLoaded` 而非 `allItems.length`：

```javascript
const [wardrobeLoaded, setWardrobeLoaded] = useState(false);

const loadWardrobe = useCallback(async () => {
    try { ... } catch { ... }
    finally { setWardrobeLoaded(true); }  // 无论成功失败都标记完成
}, []);

useEffect(() => {
    if (wardrobeLoaded) {
      setLoading(false);
      loadOutfit(selectedDate);
    }
}, [wardrobeLoaded, selectedDate]);
```

待验证（清空衣柜后进入页面）。

## 第 5 轮：PanResponder 拖拽闭包陈旧

### 原因分析
`OutfitCanvas.jsx` 中 `CanvasElement` 使用 `React.memo`，`PanResponder.create()` 在 `useRef` 中创建，但闭包捕获的是**创建时刻**的 `element.x/y`。当元素位置由外部更新（如另一元素被拖拽）时，PanResponder 内部的 `element.x` 仍是旧值。

### 修复
**OutfitCanvas.jsx** — 新增 `elementRef`，在 PanResponder 回调中通过 `elementRef.current` 读取最新值：

```javascript
const elementRef = useRef(element);
elementRef.current = element;  // 每次渲染更新

const panResponder = useRef(
    PanResponder.create({
      onPanResponderGrant: () => {
        const el = elementRef.current;  // ← 最新值
        startPos.current = { x: el.x, y: el.y };
        onSelect(el.id);
      },
      onPanResponderMove: (_, gestureState) => {
        const el = elementRef.current;  // ← 最新值
        const newX = startPos.current.x + gestureState.dx;
        const newY = startPos.current.y + gestureState.dy;
        onUpdate(el.id, { x: newX, y: newY });
      },
    })
).current;
```

## 第 8 轮：保存预览转圈 + 导出失败 — canvasRef 空引用

### 测试操作
- 点 💾 → 保存弹窗中预览图一直加载转圈

### 测试结果 ❌
- 预览图 `ActivityIndicator` 永远不消失
- 导出静默失败

### 原因分析
`OutfitCanvas.jsx` 中：
```js
const canvasRef = useRef(null);
useImperativeHandle(ref, () => canvasRef.current);
return <View ref={ref} ...>;  // ← ref 是 forwarded ref，canvasRef 从未被赋值
```
`canvasRef.current` 始终为 `null`。`captureRef(null)` / `useImperativeHandle(ref, () => null)` → 外层 `canvasRef.current` 也是 `null`。

### 修复

```javascript
const setCanvasNodeRef = useCallback((node) => {
  canvasRef.current = node;
  if (typeof ref === 'function') ref(node);
  else if (ref) ref.current = node;
}, [ref]);
return <View ref={setCanvasNodeRef} ...>;
```

## 第 10 轮：人脸相册导入不弹出 — Modal 动画冲突

### 测试操作

### 测试结果 ❌

### 原因分析
`showFaceSheet` Modal 的 `animationType="slide"` 下滑动画需要 300-400ms。原 `setTimeout(100ms)` 不够，iOS ViewController 认为"转场未完成"直接拦截了 `launchImageLibrary`。

1. **时序**: `setTimeout` 100ms → 500ms
2. **架构**: 改用 `pendingFaceRef` + `useEffect([showFaceSheet])` 模式，确保 Modal `visible=false` 状态已提交后才触发
3. **权限**: 拒绝时弹明确 Alert `"请在手机系统设置中为 Expo Go 开启相机/相册权限"`

图片数据永久保存在数据库 TEXT 字段中。

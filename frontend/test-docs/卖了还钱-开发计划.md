# 卖了还钱 — 开发计划

## 功能概述

转卖管理中心。展示所有标记为"不想要"的衣物，管理转卖状态，统计预期回血金额。

## 数据基础

- `wardrobe_items` 已有字段:
  - `is_unwanted` — 标记是否不想要 (0/1)
  - `purchase_amount` — 购入价格
  - 通过智能管家或手动可以标记 "不想要"
- `GET /api/wardrobe?is_unwanted=1` — 查询不想要的衣物
- `PUT /api/wardrobe/<id>` — 更新 (可设 is_unwanted=0 移回衣柜)
- 目前**无**转卖价、转卖状态、售出日期字段

### 需要后端新增

| # | 字段 | 类型 | 用途 |
|---|------|------|------|
| 1 | `resell_price` | REAL | 挂售/卖出价格 |
| 2 | `resell_status` | TEXT | 待售/议价中/已售出 |
| 3 | `resell_date` | TEXT | 售出日期 |

**可选方案**: 不新增字段，用 `notes` 文本存储 JSON 或自由格式记录转卖信息。MVP 先走这条免改数据库。

## 状态机

```
不想要(item.is_unwanted=1)
  └─ 待售 → 标价(记入notes) → 议价中 → 已售出(记售价+日期) → 从列表隐藏
                                                   ↘ 移回衣柜(is_unwanted=0)
```

## 关键状态

```js
unwantedItems: Item[]     // is_unwanted=1 的物品
soldItems: Item[]         // 已售出(is_unwanted=1 + notes含sold标记)
stats: {
  totalPotential: number  // 所有待售预期回血
  totalSold: number       // 已售出总金额
  count: number           // 待售数量
  soldCount: number       // 已售数量
}
```

## 交互

1. 进入页面 → GET `/api/wardrobe?is_unwanted=1` → 展示列表
2. 点击物品 → 展开详情 (图片/名称/购入价/穿着次数)
3. [标价] → 弹窗输入转卖价格，记录到 notes
4. [已售出] → 记录售价和日期到 notes，从待售列表移到已售区
5. [移回衣柜] → PUT `/api/wardrobe/<id>` is_unwanted=0 → 刷新
6. 顶部统计条实时计算预期回血总额

## MVP 范围

- 列表展示 (不新增数据库字段，用 notes 存 JSON)
- 标价功能
- 标记售出
- 移回衣柜
- 统计回血金额
- 已售记录

## 不在本次范围

- 转卖平台对接 (闲鱼等)
- 照片美化 / 文案生成
- notes JSON 格式需与智能管家兼容

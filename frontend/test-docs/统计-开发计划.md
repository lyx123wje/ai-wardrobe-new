# 统计 — 开发计划

## 功能概述

衣柜数据统计看板。CPW排行、分类占比、穿着频次分析。

## 数据基础

- `GET /api/wardrobe/stats` — 已有后端统计接口
  - 返回: `{ stats: { total_items, total_value, total_wears, cpw_ranking[], category_breakdown[] } }`
- `GET /api/wardrobe` — 全部衣物列表（前端可自行计算派生统计）

## 组件

| 文件 | 用途 |
|------|------|
| `app/statistics.jsx` | 主页面 — 总览+分类+排行+警告 (~350行) |
| `src/components/ProgressBar.jsx` | 横向进度条（分类占比用）(~40行) |

## 关键状态

```js
stats: {
  total_items, total_value, total_wears,
  cpw_ranking: [{id, sub_tag, category, cpw, wear_count, ...}],
  category_breakdown: [{category, count, value, color}],
  most_worn: [{id, sub_tag, wear_count}],
  never_worn: [{id, sub_tag}],
  high_cpw: [{id, sub_tag, cpw}],
}
```

## 数据来源

- 优先调 `GET /api/wardrobe/stats` 获取 CPW + 分类
- 补调 `GET /api/wardrobe` 获取全量做清洗（找出从未穿过、闲置等）
- 分类颜色从 `constants.js` 的 `CATEGORY_COLORS` 取

## 交互

1. 进入页面 → 两个API并行 → 组装数据 → 渲染
2. 下拉刷新
3. 纯展示无编辑（不修改数据）

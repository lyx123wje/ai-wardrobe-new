## 实现内容

### 设计思路
"视图过滤"模式 — 复用衣柜的数据模型和 API，过滤 `is_dirty=1` 的衣物形成脏衣列表。

### 技术选型
| 决策 | 原因 |
|------|------|
| 复用 `fetchWardrobe({ is_dirty: 1 })` | 和衣柜同数据源，只加查询参数 |
| 单列列表 | 脏衣服通常不多，列表比网格更清晰 |
| Pressable（不用 Reanimated） | 避免原生层 C++ 冲突，保证稳定 |
| `useSafeAreaInsets()` | 适配刘海屏，不用硬编码 padding |
| 乐观更新 | 先移出列表再调 API，手感秒快，失败回滚 |
| 复用 DetailModal | 和衣柜共享查看/编辑/删除组件 |

### 功能清单

| 功能 | 实现方式 |
|------|----------|
| 脏衣列表 | FlatList 单列 + 缩略图 + 名称 + 分类徽章 + 颜色 |
| 顶部统计 | 红色圆角徽章显示件数 |
| 单件清洗 | 点绿色"已清洗"按钮，乐观移除 + API 调用 |
| 一键全洗 | 底部"全部洗好了 (N 件)"按钮，确认弹窗 + `markAllClean()` |
| 点击详情 | 复用 DetailModal，支持查看/编辑/删除/标记 |
| 下拉刷新 | RefreshControl 刷新脏衣数据 |
| 空状态 | "暂无脏衣，真干净！" |
| 图片加载 fallback | 灰色占位块 + "?" |

# AI 衣橱后端服务说明书

## 一、项目概述

本项目是一个基于 Flask 的 AI 衣物处理后端服务，v3.0 包含三大引擎：
- **Clothing Engine**: 衣物去背景 + MobileNetV2 分类（7 大类 + 104 细粒度标签）
- **Portrait Engine**: 人像抠图（u2net_human）+ 发型素材管理
- **Mind Engine**: 角色人格映射 + DeepSeek LLM 思维推理

### 3.2 图片去背景功能

使用 rembg 库的 `remove()` 函数实现 AI 去背景：
```python
transparent_data = remove(img_data)
```
- 输入：原始图片二进制数据
- 输出：透明背景的 PNG 图片二进制数据
- 首次运行会自动下载 u2net 预训练模型

### 3.3 衣物分类功能（AI 真实版）

使用 MobileNetV2 预训练模型实现真实的 AI 图像分类：

```python
import torch
from torchvision import models, transforms

# 加载轻量级 MobileNetV2 预训练模型（约 45MB）
# 兼容新旧版本 torchvision
try:
    model = models.mobilenet_v2(weights=models.MobileNetV2_Weights.DEFAULT)
except AttributeError:
    model = models.mobilenet_v2(pretrained=True)
model.eval()  # 设置为评估模式
```

**图片预处理流程：**
1. Resize 到 256x256
2. 中心裁剪为 224x224（模型输入标准尺寸）
3. 转换为 Tensor
4. 归一化处理（使用 ImageNet 标准均值和标准差）

**AI 推理：**
```python
with torch.no_grad():  # 禁用梯度计算，加速推理
    output = model(input_tensor)
```

### 3.4 层级映射表设计

**核心概念：** 将 ImageNet 的 1000 细粒度类别映射到衣橱层级结构

**映射表结构：**
```python
CLOTHING_MAP = {
    # 索引: (小标签名称, 大类归属)
    608: ("牛仔裤", "裤子"),
    617: ("长袖衬衫", "上衣"),
    836: ("卫衣", "上衣"),
    413: ("运动鞋", "鞋子"),
    # ... 共约 104 个衣物相关类别
}
```

**支持的大类：**
| 大类 | 包含小标签示例 |
|------|--------------|
| 👕 上衣 | T恤、衬衫、卫衣、毛衣、夹克、西装等 |
| 👖 裤子 | 牛仔裤、休闲裤、运动裤、睡裤等 |
| 👗 裙子 | 连衣裙、半身裙、长裙、婚纱等 |
| 👟 鞋子 | 运动鞋、高跟鞋、靴子、凉鞋等 |
| 🧢 帽子 | 棒球帽、渔夫帽、针织帽等 |
| 👜 包包 | 手提包、背包、钱包等 |
| 🎀 其他 | 围巾、领带、首饰等 |

**分类查询函数：**
```python
def get_clothing_info(class_idx):
    # 1. 优先查找映射字典
    info = CLOTHING_MAP.get(class_idx)
    if info:
        return info[0], info[1], class_idx
    
    # 2. 未映射但在 ImageNet 范围内，返回原始英文名
    if 0 <= class_idx < len(IMAGENET_LABELS):
        raw_name = IMAGENET_LABELS[class_idx]
        return raw_name, "待分类", class_idx
    
    # 3. 完全未知
    return "未知单品", "未分类", class_idx
```

**工程折中方案：**
- **ImageNet（通用模型）：** 能识别 1000 种物体，但衣物分类较细
- **DeepFashion（垂直模型）：** 专门针对衣物训练，但更复杂
- **当前方案：** 通过映射表维护分类关系，成本最低且效果可控

### 3.6 API 接口实现

**接口地址：** `POST /api/process`

**请求参数：**
- `image` - 图片文件（multipart/form-data）
- 或 `image_base64` - Base64 编码的图片（JSON 格式）

**响应格式（v2.0 升级）：**
```json
{
    "status": "success",
    "sub_tag": "牛仔裤",           // AI 识别的细粒度小标签
    "category": "裤子",            // 自动归类的大类
    "raw_index": 608,             // ImageNet 原始索引（用于排错）
    "processed_image_base64": "data:image/png;base64,..."
}
```

**错误处理：**
- 400 - 未找到图片字段 / 文件名为空
- 500 - 服务器内部错误

## 五、API 接口文档

### 5.1 Clothing Engine 接口

**POST /api/process** - 衣物抠图+分类

请求参数：
- `image` - 图片文件（multipart/form-data）
- 或 `image_base64` - Base64 编码的图片（JSON 格式）

响应格式：
```json
{
    "status": "success",
    "sub_tag": "卫衣",
    "category": "上衣",
    "raw_index": 836,
    "processed_image_base64": "data:image/png;base64,..."
}
```

### 5.2 Portrait Engine 接口

**POST /api/process_portrait** - 人像抠图

使用 u2net_human 模型专门优化人体分割，请求参数同 /api/process。

响应格式：
```json
{
    "status": "success",
    "processed_image_base64": "data:image/png;base64,..."
}
```

**GET /api/hairstyles/list** - 发型文件列表

读取 static/hairstyles/ 目录下的 .png 文件。

响应格式：
```json
{
    "hairstyles": ["bob.png", "long.png", "pixie.png"]
}
```

### 5.3 Mind Engine 接口

**GET /api/personas/list** - 角色人格列表

返回所有 6 个角色人格数据。

**POST /api/personas/match** - 衣物→角色匹配

请求体：
```json
{
    "clothing_tag": "卫衣",
    "clothing_category": "上衣"
}
```

响应格式：
```json
{
    "status": "success",
    "matches": ["streetwear_kid", "minimalist_master"],
    "matched_by": "tag",
    "personas": [...]
}
```

匹配策略：
1. 先通过 associated_tags 精确匹配 sub_tag
2. 无精确匹配时，通过 associated_categories 大类兜底
3. 仍无匹配则返回全部角色

**POST /api/persona_think** - 角色思维推理（LLM）

请求体：
```json
{
    "persona_id": "streetwear_kid",
    "user_problem": "周末逛街穿什么？",
    "clothing_tag": "卫衣"
}
```

响应格式：
```json
{
    "status": "success",
    "persona_name": "街头潮人",
    "response_text": "嘿！卫衣是街头穿搭的灵魂单品...",
    "error": null
}
```

需要配置 DEEPSEEK_API_KEY 后才能正常返回 LLM 响应。

## 七、常见问题与解决方案

### 7.1 torchvision 版本兼容问题

**错误信息：**
```
AttributeError: module 'torchvision.models' has no attribute 'MobileNetV2_Weights'
```

**原因：** 不同版本 torchvision 的 API 写法不同

**解决方案：** 使用 try-except 兼容新旧版本写法

---

### 7.2 分类不准确，总是显示"上衣"

**现象：** AI 识别索引不在映射表中，走到默认返回值

**调试方法：**
查看控制台输出：
```
AI 识别原始索引: 697
```

**修正方法：**
1. 记录控制台输出的原始索引（如 697）
2. 查询该索引对应的 ImageNet 类别含义
3. 将索引添加到对应的分类映射中：
```python
CLOTHING_MAP[697] = ("睡裤", "裤子")
```

**持续优化流程：**
1. 上传图片测试
2. 查看控制台输出的 `AI 识别原始索引: xxx`
3. 将 xxx 添加到对应的分类映射
4. 刷新页面重新测试

---

### 7.3 未映射类别处理

**现象：** AI 识别到不在映射表中的类别

**处理策略：**
- 返回原始英文名称 + "待分类" 大类
- 保留 `raw_index` 字段便于后续分析
- 后台记录未映射索引，定期补充映射表
